# 上电当天30分钟执行单

目标：在最短时间内完成板卡上电、基础联通、FPGA到PC链路验证。

## 0-5分钟：硬件与供电确认
1. 连接板卡电源并上电，确认 PWR 指示灯常亮。
2. 连接 USB 到电脑，确认下载工具可识别板卡。
3. 暂不接摄像头，先使用板端固定测试帧或测试图像流。

通过标准：板卡稳定上电，不反复掉线。

## 5-10分钟：网络连通确认
1. 连接板卡网口与电脑或交换机。
2. 配置板卡 IP，例如 192.168.1.10。
3. 配置电脑网卡到同网段，例如 192.168.1.100。
4. 在电脑执行 ping 192.168.1.10。

通过标准：ping 连续成功，无明显丢包。

## 10-15分钟：协议参数对齐
1. 打开协议文档，确认字段顺序、字节序、长度、CRC。
2. 确认板端发送参数：
- magic: 0x45544C50
- version: 1
- header_len: 40
- payload_len: width * height * channels
3. ROI 暂时可固定或置零，先确保链路稳定。

参考：doc/fpga_pc_protocol.md

通过标准：板端发送内容与协议一致。

## 15-20分钟：PC端接收参数配置
1. 修改配置文件中的主机和端口。
2. host 填板卡 IP，port 填板端发送端口。
3. 保持 source 使用 fpga_tcp。

参考：pc/config/default.yaml

通过标准：配置与板端一致。

## 20-25分钟：运行接收与可视化
1. 在 pc 目录运行上位机程序。
2. 观察控制台是否持续处理帧。
3. 若启用显示窗口，观察画面是否持续刷新。

通过标准：Processed frames 持续增长，程序不报协议错误。

## 25-30分钟：结果验证与问题定位
1. 检查结果文件是否生成并持续写入。
2. 若失败，按优先级排查：
- 供电不稳
- IP 不通
- 端口不一致
- magic 错误
- payload_len 或 CRC 错误
3. 记录本次联调参数和现象，便于下次快速复现。

结果文件：results/run_result.csv

## 常用命令
在项目根目录运行：
- 本地模拟链路：.\scripts\run_mock_link.ps1 -Show

在 pc 目录运行：
- 真实板卡接收：python -m app.main --config config/default.yaml --source fpga_tcp --show

## 成功判据
1. 能稳定接收至少300帧。
2. 无 magic、payload_len、CRC 报错。
3. 结果文件持续输出。
4. 程序可重复运行并得到一致行为。
